## v6.0 关键设计原则

1. **框架无关**：v6.0 应该可以移植到 Hermes/Codex/LangChain 等
2. **可观测**：暴露 Prometheus / OpenTelemetry metrics
3. **可扩展**：plugin 机制（第三方可加专科）
4. **可协作**：多实例（每个 Agent 一个 doctor 实例）

## v5.0 已知局限（为 v6.0 留的 TODO）

| 局限 | 影响 | v6.0 解法 |
|------|------|----------|
| PCEC 只能感知本机文件 | 跨主机问题查不到 | v6.2 跨主机协同 |
| 药方库手工维护 | 跟不上新框架 | v6.1 自动从 GitHub issues 学 |
| 8 个专科是预定义 | 细粒度问题覆盖不全 | v6.3 plugin 机制 |
| 静默规则硬编码 | 不同用户需求不同 | v6.0 配置化 |

## 18 个月扩展路径
